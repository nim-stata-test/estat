1. Study Rationale

Why This Study?

This intervention study tests whether adjusting heat pump scheduling and heating curve parameters can improve energy efficiency (COP) while maintaining thermal comfort. The four strategies represent different trade-offs between comfort, energy consumption, solar self-sufficiency, and electricity costs.

Historical data analysis (Phases 1-4) suggests potential COP improvements of 0.2-0.4 through schedule optimization, but these predictions need validation under real operating conditions with varying weather, occupancy, and system dynamics.

Crossover Design

This study uses a randomized crossover design where each strategy is tested multiple times throughout the heating season. Key design features:

  • Within-subject comparison: The same building tests all strategies, eliminating between-building variability
  • Seasonal balance: Each strategy appears in early, mid, and late winter to account for changing outdoor temperatures and solar availability
  • Randomization: Block order is randomized to prevent systematic bias from time trends
  • Washout periods: 3-day washout between strategies allows the thermal mass to equilibrate to new settings before measurement
  • Replication: ~9 blocks per strategy provides increased statistical power to detect meaningful differences

Block Structure

Each 5-day block consists of:

  • Days 1-3 (Washout): System adjusts to new parameters. Indoor temperatures stabilize. Data from this period is excluded from primary analysis.
  • Days 4-5 (Measurement): System operates in steady-state. COP, energy consumption, and comfort metrics are recorded for analysis.

The 3-day washout was determined from historical data showing the building's thermal time constant is approximately 14-20 hours. Three days provides >99% equilibration.

7. Daily Checklist

At Block Start (Day 1)

  • Verify current block number and strategy from schedule
  • Update heat pump comfort schedule (start/end times)
  • Update Home Assistant climate setpoints (comfort/eco)
  • Update heating curve rise (Steilheit) on heat pump
  • Record parameter change timestamp in daily log
  • Note any unusual conditions (weather events, occupancy changes)

During Washout (Days 1-3)

  • Monitor indoor temperatures stabilizing to new settings
  • Check heat pump is responding to new schedule
  • Note any error conditions or anomalies

During Measurement (Days 4-5)

  • Verify system is operating in steady-state
  • Check data logging is active (Home Assistant, InfluxDB)
  • Record daily COP and energy consumption (optional manual check)
  • Note weather conditions and any disruptions

At Block End (Day 5)

  • Complete block summary entry
  • Export any manual observations
  • Prepare for next block's parameter changes

8. Handling Missed Block Changes

What Constitutes a "Missed" Block Change?

A block change is considered missed if parameters were not updated within 24 hours of the scheduled block start date. Minor delays (a few hours) do not invalidate a block, but significantly late changes affect the washout period and may compromise measurement validity.

Scenario 1: Change Delayed by 1 Day

Situation: Parameters changed on Day 2 instead of Day 1.

Impact: Washout period is shortened by 1 day.

Action:

  • Continue with the block as scheduled
  • Document the delay in the block notes
  • Mark the block as "shortened washout" in the daily log
  • Include in analysis with a sensitivity flag

Scenario 2: Change Delayed by 2+ Days

Situation: Parameters changed on Day 3 or later.

Impact: Washout period is severely compromised or eliminated.

Action:

  • Implement the change immediately when discovered
  • Document the delay with exact timestamps
  • Mark the block as "excluded" in the block summary
  • Do NOT attempt to extend the block or shift subsequent blocks
  • Continue with the next scheduled block on its original date

Scenario 3: Entire Block Missed

Situation: No parameter change was made during the entire block period.

Impact: Block continues previous strategy settings.

Action:

  • Mark the block as "missed" in the block summary
  • Document what settings were actually in effect
  • In analysis, this block's data will be attributed to whatever strategy was actually running
  • Begin the next block on schedule with correct parameters

Scenario 4: Wrong Strategy Implemented

Situation: Parameters were changed, but to the wrong strategy.

Impact: Block data is misattributed.

Action:

  • If discovered during washout (Days 1-3): Correct immediately and note the error
  • If discovered during measurement (Days 4-5): Complete the block with current settings
  • Document which strategy was actually implemented
  • In analysis, attribute data to the actual strategy that was running

Key Principles

  • Never shift the schedule: Subsequent blocks always start on their original dates. Shifting creates cascading problems and breaks seasonal balance.
  • Document everything: Detailed notes allow proper handling during analysis. A well-documented deviation is better than an undocumented one.
  • Intention-to-treat: The primary analysis will use the intended schedule. Sensitivity analyses will examine actual-strategy effects.
  • No makeup blocks: Do not add extra blocks to compensate for missed ones. The design has sufficient replication to handle occasional missing data.
