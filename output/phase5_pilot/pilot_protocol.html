Purpose

This pilot experiment explores the heating parameter space using a T_HK2-targeted design. The design maximizes spread in flow temperature (T_HK2) rather than raw parameters, because T_HK2 is what drives the thermal response we need to learn.

Two Analysis Approaches

Approach Data Used Best For
Dynamical (preferred) All 7 days (~6,700 points) Learning dynamics, model validation
RSM Days 3-7 (10 block means) Simple parameter effects

The dynamical analysis uses the grey-box state-space model and treats transitions between settings as informative step responses (no washout needed).

Safety Constraints
  • Minimum T_weighted: 17.0°C - abort block if breached
  • Minimum COP: 2.0 - check heat pump if sustained below
  • Maximum violation %: 50% (exploration allows more than Phase 5)

4. Daily Checklist

At Block Start (Day 1 of each block)

  • Update heat pump comfort schedule (start/end times)
  • Update Home Assistant climate setpoints (comfort/eco)
  • Update heating curve rise (Steilheit) on heat pump
  • Record parameter change timestamp
  • Note current outdoor temperature and weather

During Measurement (Days 3-7)

  • Check T_weighted stays above 17.0°C
  • Monitor COP remains reasonable (>2.0)
  • Note any occupancy changes or unusual events
Abort Criteria

Revert to baseline settings immediately if:

  • T_weighted drops below 16.5°C for more than 2 hours
  • COP remains below 2.0 for more than 24 hours
  • Heat pump shows error codes

Baseline: comfort=20.2°C, eco=18.0°C, curve=0.97, schedule=06:30-20:00
